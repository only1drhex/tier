Guest Question.

Given a z.o.h. in cascade with \(G_{1}(s) = (s + 2) /( s+ 1) \) or

\(G\left ( s \right ) = \frac{1-e^{-Ts}}{s}\frac{\left ( s+2 \right )}{\left ( s+1 \right )} \) (13:38)

find the sampled-data transfer function, G(z), if the sampling time, T, is 0.5 second.

Admin
Solution

Equation (13.38) represents a common occurrence in digital control systems, namely a transfer function in cascade with a zero-order hold. Specifically, \(G_{1}(s) = (s + 2) /( s+ 1) \) is in cascade with a zero-order hold, ( \( 1 – e^{-Ts} / s \)). We can formulate a general solution to this type of problem by moving the s in the denominator of the zero-order hold to \(G_{1}(s) \) , yielding

\(G\left ( s \right ) = (1-e^{-Ts}) \frac{G_{1}(s)}{s} \) (13:39)

from which

\(G\left ( z \right ) = (1-z^{-1})z\left \{\frac{G_{1}(s)}{s} \right \} = \frac{z-1}{z}z\left \{\frac{G_{1}(s)}{s} \right \} \) (13:40)

Thus, begin the solution by finding the impulse response (inverse Laplace transform) of \(G_{1}(s) \)/s. Hence,

\(G_{2}\left ( s \right ) = \frac{G_{1}(s)}{s} = \frac{s+2}{s(s+1)} = \frac{A}{s}+\frac{B}{s +1} =\frac{2}{s}-\frac{1}{s +1}\) (13:41)

Taking the inverse Laplace transform, we get

\(g_{2}\left ( t \right )=2-e^{-t}\) (13:42)

from which

\(g_{2}\left ( kT \right )=2-e^{-kt}\) (13:43)

Using Table 13.1 below , we find

\(G_{2}\left ( z \right ) = \frac{2z}{z-1} – \frac{z}{z-e^{-T}}\) (13:44)

Substituting T = 0.5 yields

\(G_{2}\left ( z \right ) = z\left \{\frac{G_{1}\left ( s \right )}{s} \right \} =\frac{2z}{z-1} – \frac{z}{z-0.607} =\frac{z^{2}- 0.213z}{\left ( z-1 \right )\left ( z-0.607 \right )}\) (13:45)

From Eq. (13.40)

\(G\left ( z \right ) = \frac{z-1}{z}G_{2}\left ( z \right )= \frac{z-0.213}{z-0.607}\) (13:46)

We can convert G1(s) in cascade with a zero-order hold (z.o.h.) to G(z) using MATLAB’s G=c2d (G1, T,’zoh’) command, where G1 is an LTI continuous-system object and G is an LTI sampled-system object. T is the sampling interval and ’zoh’is a method of transformation that assumes G1(s) in cascade with a z.o.h. We simply put G1(s) into the command (the z.o.h. is automatically taken care of) and the command returns G(z).
